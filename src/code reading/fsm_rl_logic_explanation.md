# FSMState_RL.cpp 逻辑详解

## 文件概述

这是 TITA 机器人 **强化学习（RL）运动控制状态** 的实现文件。它使用 TensorRT CUDA 运行神经网络策略，实现端到端的轮腿机器人运动控制。

---

## 核心数据结构

```cpp
// 观测维度 (31维)
input_0[31]   // 当前观测
input_1[310]  // 历史缓冲区 (31维 * 10帧)
output[8]     // 网络输出 (8个关节的目标位置偏移)

// 关节索引映射
索引 0-2: 左腿关节 (髋、大腿、小腿)
索引 3:   左轮 (轮子关节)
索引 4-6: 右腿关节 (髋、大腿、小腿)  
索引 7:   右轮 (轮子关节)
```

## 主要函数流程

### 1. 构造函数 `FSMState_RL()`

```
┌─────────────────────────────────────────┐
│  初始化 CUDA TensorRT 引擎               │
│  cuda_test_("/your/path/to/xx.engine")  │
├─────────────────────────────────────────┤
│  分配内存:                               │
│  - input_0[31]   (当前观测)              │
│  - input_1[310]  (10帧历史)              │
│  - output[8]     (网络输出)              │
│  - output_last[8](上一动作缓存)          │
│  - input_1_temp[279] (滑动窗口临时)      │
└─────────────────────────────────────────┘
```

---

### 2. 进入状态 `enter()`

```
┌─────────────────────────────────────────┐
│ Step 1: 记录轮子初始位置                 │
│ wheel_init_pos_abs_[i] = q[4*i+3]       │
│ (索引 3 和 7，即两个轮子)                │
├─────────────────────────────────────────┤
│ Step 2: 初始化期望位置                   │
│ desired_pos[] = 当前关节位置             │
├─────────────────────────────────────────┤
│ Step 3: 填充 obs_ 结构体                 │
│ - dof_pos[8]: 关节位置                   │
│ - dof_vel[8]: 关节速度                   │
├─────────────────────────────────────────┤
│ Step 4: 设置模型参数                     │
│ - action_scale = 0.5                     │
│ - lin_vel_scale = 2.0                    │
│ - ang_vel_scale = 0.25                   │
│ - dof_pos_scale = 1.0                    │
│ - dof_vel_scale = 0.05                   │
├─────────────────────────────────────────┤
│ Step 5: 默认关节位置                     │
│ default_dof_pos = {0, 0.8, -1.5, 0,      │
│                    0, 0.8, -1.5, 0}      │
│ (索引 3 和 7 是轮子，默认值为 0)         │
├─────────────────────────────────────────┤
│ Step 6: 清空历史缓冲区                   │
│ input_1[310] = 0                         │
│ output_last[8] = 0                       │
├─────────────────────────────────────────┤
│ Step 7: 预热填充 (关键!)                 │
│ for i in range(10):                      │
│   _GetObs()                              │
│   滑动窗口更新 input_1                   │
│ (确保历史缓冲区有有效数据)               │
├─────────────────────────────────────────┤
│ Step 8: 预热前向传播                     │
│ for i in range(10):                      │
│   _Forward()                             │
├─────────────────────────────────────────┤
│ Step 9: 启动后台推理线程                 │
│ forward_thread = thread(_Run_Forward)    │
└─────────────────────────────────────────┘
```

---

### 3. 主控制循环 `run()`

**调用频率**: 与控制器主循环相同 (通常 500Hz)

```
┌─────────────────────────────────────────┐
│ Step 1: 读取遥控命令                     │
│ x_vel_cmd_  = twist_linear[X]            │
│ y_vel_cmd_  = twist_linear[Y]            │
│ pitch_cmd_  = twist_angular[Z]           │
├─────────────────────────────────────────┤
│ Step 2: 清零命令缓冲区                   │
│ qd = 0, qd_dot = 0                       │
│ kp = 0, kd = 0, tau_cmd = 0              │
├─────────────────────────────────────────┤
│ Step 3: 计算力矩输出 (PD控制)            │
│                                          │
│ 轮子关节 (i%4==3, 即索引 3,7):           │
│   tau_cmd[i] = 12 * desired_pos[i]       │
│              + 1.5 * (0 - dq[i])         │
│                                          │
│ 腿部关节 (其他索引):                     │
│   tau_cmd[i] = 60 * (desired_pos[i] - q[i])
│              + 2.0 * (0 - dq[i])         │
│                                          │
│ 注意: desired_pos 由后台线程更新         │
└─────────────────────────────────────────┘
```

**关键点**: `run()` 只负责执行 PD 控制，神经网络推理在后台线程完成。

---

### 4. 观测构建 `_GetObs()`

**输出**: 31维观测向量 input_0

```
┌───────────────────────────────────────────────────┐
│ [0-2] 角速度 (3维)                                 │
│ angvel = 0.97*omegaBody + 0.03*a_l  (低通滤波)    │
│ obs = angvel * 0.25                                │
├───────────────────────────────────────────────────┤
│ [3-5] 重力投影 (3维)                               │
│ _B2G_RotMat = state_estimator->rBody              │
│ projected_gravity = _B2G_RotMat * (0,0,-1)        │
├───────────────────────────────────────────────────┤
│ [6-8] 命令 (3维)                                   │
│ vel_x = x_vel_cmd_ * 2                            │
│ vel_y = y_vel_cmd_ * 2                            │
│ angle = pitch_cmd_ * 3.14 -> normalize -> *0.25   │
├───────────────────────────────────────────────────┤
│ [9-14] 关节位置 (6维) ⚠️ 关键修改                  │
│ 跳过轮子索引 3 和 7，只使用腿部关节                │
│ for i in 0..7:                                     │
│   if i==3 or i==7: continue                       │
│   pos = (dof_pos[i] - default_dof_pos[i]) * scale │
├───────────────────────────────────────────────────┤
│ [15-22] 关节速度 (8维)                             │
│ 包含所有关节，包括轮子                             │
│ vel = dof_vel[i] * 0.05                           │
├───────────────────────────────────────────────────┤
│ [23-30] 上一动作 (8维)                             │
│ 包含所有关节                                       │
│ last_action = output_last[i]                      │
└───────────────────────────────────────────────────┘
```

---

### 5. 单次推理 `_Forward()`

```
┌─────────────────────────────────────────┐
│ Step 1: 构建当前观测                     │
│ _GetObs() -> input_0[31]                │
├─────────────────────────────────────────┤
│ Step 2: CUDA 神经网络推理                │
│ do_inference(input_0, input_1, output)  │
│                                         │
│ 输入:                                    │
│ - input_0[31]:  当前观测                 │
│ - input_1[310]: 10帧历史                 │
│ 输出:                                    │
│ - output[8]:    8个关节的目标位置偏移    │
├─────────────────────────────────────────┤
│ Step 3: 更新历史缓冲区 (滑动窗口)        │
│                                         │
│ 旧数据左移:                              │
│ for i in 0..278:                        │
│   input_1[i] = input_1[i+31]            │
│                                         │
│ 新数据追加:                              │
│ for i in 0..30:                         │
│   input_1[i+279] = input_0[i]           │
├─────────────────────────────────────────┤
│ Step 4: 保存输出                         │
│ output_last[i] = output[i]              │
└─────────────────────────────────────────┘
```

---

### 6. 后台推理线程 `_Run_Forward()`

**运行频率**: 100Hz (每 10ms 一次)

```
┌─────────────────────────────────────────┐
│ while (threadRunning):                  │
│                                         │
│   Step 1: 读取当前关节状态               │
│   for i in 0..3:                        │
│     obs_.dof_pos[i+4] = q[i]      (右腿)│
│     obs_.dof_vel[i+4] = dq[i]           │
│     obs_.dof_pos[i]   = q[i+4]    (左腿)│
│     obs_.dof_vel[i]   = dq[i+4]         │
│   obs_.dof_pos[3] = 0   (左轮置零)      │
│   obs_.dof_pos[7] = 0   (右轮置零)      │
├─────────────────────────────────────────┤
│   Step 2: 神经网络推理                   │
│   _Forward()                            │
├─────────────────────────────────────────┤
│   Step 3: 处理网络输出                   │
│   for j in 0..7:                        │
│     if j%4==0 (轮子):                   │
│       action[j] = output[j]*0.5*0.5 + default│
│     else (腿部):                        │
│       action[j] = output[j]*0.5 + default    │
├─────────────────────────────────────────┤
│   Step 4: 更新期望位置                   │
│   for i in 0..3:                        │
│     desired_pos[i+4] = action[i]  (右腿)│
│     desired_pos[i]   = action[i+4] (左腿)│
├─────────────────────────────────────────┤
│   Step 5: 精确睡眠 10ms                  │
│   absoluteWait(_start_time, 10000us)    │
└─────────────────────────────────────────┘
```

---

### 7. 状态切换 `checkTransition()`

```cpp
switch (fsm_state_name) {
  case RECOVERY_STAND:  → 切换到恢复站立
  case RL:              → 保持当前状态
  case TRANSFORM_DOWN:  → 切换到降下/关机
  case PASSIVE:         → 切换到被动/停止
}
```

---

### 8. 退出状态 `exit()`

```
stop_update_ = true   // 停止后台线程更新
```

## 关键设计要点

1. **双线程架构**
   - 主线程 (`run()`): 高频 PD 控制 (500Hz)
   - 后台线程 (`_Run_Forward()`): 神经网络推理 (100Hz)

2. **观测设计 (31维)**
   - 角速度: 3维
   - 重力投影: 3维
   - 命令: 3维
   - 关节位置: **6维** (去掉轮子索引 3,7)
   - 关节速度: 8维
   - 上一动作: 8维

3. **历史缓冲区**
   - 保存 10 帧历史观测
   - 使用滑动窗口更新

4. **轮子特殊处理**
   - 观测中轮子位置置零
   - 轮子 PD 增益较低 (kp=12 vs 腿部 kp=60)
